Mobile App / AI / OCR

냉코 냉장고 관리 앱

냉코는 식재료 등록, 소비기한 관리, 영수증 OCR, AI 레시피 추천, 가계부 흐름을 하나로 묶은 React Native 앱입니다. 앱은 오프라인 사용성을 고려해 로컬 데이터를 중심으로 동작하고, API는 상품 조회와 AI/OCR 보강 역할을 맡습니다.

기획, 개발, App Store 출시, 운영

핵심 기능

  • Expo Router 기반 모바일 앱에서 냉장고, 식재료, 레시피, 지출 화면을 구성했습니다.
  • Expo SQLite와 Context 상태를 활용해 식재료/소비기한 데이터를 앱 내부에서 빠르게 관리합니다.
  • API는 Supabase 상품 DB, 사용자 기여 상품, 공공/외부 데이터, Gemini/OpenAI 기반 분석 흐름을 연결합니다.

Realtime Media / WebRTC

FilmMate

FilmMate는 촬영자와 모델이 서로 다른 기기를 들고 실시간 카메라 화면을 공유하며 원격 촬영할 수 있는 모바일 앱입니다. Firebase Firestore는 시그널링 저장소로 사용하고, P2P 연결이 어려운 네트워크 환경에서는 VPS 기반 TURN 중계를 사용합니다.

기획, 개발, App Store 배포

핵심 구현

  • React Native
  • Expo
  • WebRTC
  • Firebase Auth
  • Firestore
  • STUN / TURN
  • 익명 Firebase Auth로 시그널링 권한을 확보하고, Firestore `webrtcRooms` 문서에 offer, answer, ICE candidate를 교환합니다.
  • STUN은 Google 서버를 사용하고, TURN credential은 `turn-cred.filmmate.site` 엔드포인트에서 발급받도록 구성했습니다.
  • 촬영자와 모델 역할을 분리하고, 연결 후에는 카메라 미리보기, 원격 촬영, 사진 전송, 필름/인생네컷 플로우를 제공합니다.
  • 연결 실패, ICE 실패, 권한 오류, 타임아웃을 분류해 사용자에게 복구 가능한 상태로 안내합니다.
Firestore Offer/Answer/ICE signaling
VPS TURN NAT 환경 영상 중계
Camera UX 원격 촬영과 사진 공유

Mason CLI

Mason은 Safetia 모노레포 안에서 자연어 화면 요구사항을 ui-spec/flow-spec으로 정규화하고, 디자인 시스템 기반 프로덕션 코드, 실제 캡처, 화면설계서 PPTX, 작업 히스토리까지 생성하는 내부 CLI입니다.

설계, 구현, 자동화 파이프라인 구축

자동화 대상

  • Codex CLI
  • Storybook
  • Design System
  • Playwright
  • PPTX
  • QA Report
  • 한국어 요구사항을 Mason이 이해할 수 있는 화면 명세와 필드 단위 의도로 변환합니다.
  • Storybook index와 컴포넌트 레지스트리를 기반으로 ListSection, DynamicForms, Table, FileUploader 같은 공통 컴포넌트를 우선 매칭합니다.
  • 생성된 코드를 실제 렌더링하고 Playwright로 캡처한 뒤, 번호 설명과 화면설계서 PPTX를 생성합니다.
  • 캡처 누락, 빈 이미지, 마커 겹침, callout 누락, 중복 버튼 등을 QA 리포트로 검증합니다.

F-Lab 기업회원 채용 제휴 서비스

F-Lab 멘토링 플랫폼 안에서 기업회원이 멘티 이력서를 탐색하고, 후보자를 관리하며, 채용전형을 만들어 제안까지 보낼 수 있는 B2B 채용 제휴 서비스를 구현했습니다. 프론트는 홈페이지, 백오피스, LMS 패키지에 걸쳐 있고, 백엔드는 Kotlin/Spring business 모듈로 가입/검수/이력서/제안 흐름을 분리했습니다.

  • React
  • Recoil

  • Spring
  • JPA
  • 기업회원 가입 폼, 사업자번호/담당자 검증, 기업 권한 확인, 서비스 페이지 접근 제어를 구현했습니다.

  • 백오피스에서는 기업회원 신청 목록, 상세 폼 조회, 승인/거절 처리를 제공하고, LMS에서는 멘티가 받은 제안을 확인하고 수락/거절할 수 있게 연결했습니다.

Kakao Map / Exhaustive Search

Safetia 카카오맵 격자 완전 탐색

Safetia 지도 기능은 사업장 주소를 기준으로 주변 시설을 탐색하고, 카카오 Places API의 페이지/검색 한계에 도달하면 검색 영역을 사각 격자로 재귀 분할해 누락 가능성을 줄이는 지도 기반 데이터 수집 도구입니다.

지도 검색 알고리즘, 일괄 갱신, Excel 운영 도구 구현

핵심 구현

  • Next.js
  • TypeScript
  • Kakao Map API
  • Kakao Local API
  • Recoil
  • ExcelJS
  • 주소를 Kakao Geocoder로 좌표화하고, 중심 좌표 기준 3km 반경의 시설 데이터를 수집합니다.
  • Places API 결과가 페이지 한계에 걸리면 검색 영역을 사각 bounds로 전환하고, rect를 4분할해 각 격자를 다시 탐색합니다.
  • 격자별 결과는 장소 ID와 키워드 기준으로 중복 제거하고, 중심점과의 거리 계산을 통해 1km/2km/3km range로 분류합니다.
  • 사업장 일괄 갱신, API 사용량 제한, 실패 주소 목록, 엑셀 다운로드 흐름을 운영 도구로 연결했습니다.

Excel 운영용 일괄 갱신/다운로드

Snapshoot 스마트폰 렌탈 서비스

Snapshoot은 SNS 기반 스마트폰 렌탈 서비스를 웹, API, 관리자 시스템으로 분리해 운영한 풀스택 프로젝트입니다. Next.js 사용자 웹과 관리자 웹, NestJS API, AWS EB/EC2, S3, RDS를 연결해 주문과 운영 대응 흐름을 구축했습니다.

기획, FE, BE, 인프라, 운영

시스템 구성

  • Next.js
  • NestJS
  • AWS EB / EC2
  • S3
  • RDS
  • 사용자 웹은 주문, 인증, 결제, 다국어, 파트너/가이드 페이지를 담당합니다.
  • NestJS API는 주문, 재고, 락커, 반납, 배너, 쿠폰, 포인트, 결제 연동 등 운영 도메인을 처리합니다.
  • 관리자 웹은 주문/재고/락커/메시지/쿠폰 관리와 운영 이슈 대응 도구 역할을 합니다.
  • API 예외와 결제 잠금 타임아웃 등 운영 이슈는 Slack 알림으로 빠르게 확인할 수 있게 구성했습니다.
